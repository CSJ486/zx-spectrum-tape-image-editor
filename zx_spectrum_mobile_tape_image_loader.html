User Manual Help & Guides
1. How to Code in Sinclair BASIC +

Type normal Sinclair BASIC with a number at the start of each line. The compiler uses those line numbers to build a valid program listing.

  • Start every instruction with a line number such as 10, 20, 30.
  • Use PRINT, CLS, LOAD, PAUSE, GO TO, and other keywords as usual.
  • Keep strings inside quotes. Example: 10 PRINT "HELLO".
  • For image loaders, a common starter line is LOAD "" SCREEN$.

The BASIC Editor automatically tokenizes supported Sinclair keywords during compilation.

2. Adding an Image to your BASIC Program +

Use Select PHOTO / IMAGE inside the BASIC Editor to import a picture and convert it into tape screen data.

  • Choose an image file from your device.
  • Select the fit mode: Crop, Smart Contain, Pad, or Stretch.
  • Keep Append Converted Image (Screen$) enabled when you want the image included in the TAP.
  • If you want the picture to show after loading, include a loader line such as LOAD "" SCREEN$ in the BASIC program.

This adds the converted image as Spectrum screen data so the tape can load both code and picture together.

3. Saving & Exporting to Tape (.TAP / .WAV) +

The app can export your program as a .TAP file and also create a .WAV tape audio version.

  • .TAP is the tape block file used by Spectrum emulators.
  • .WAV is useful when you want a real audio cassette-style file.
  • Tap Compile to .TAP in the BASIC Editor to build the program tape.
  • Tap Export Current Tape to .WAV in Tape Play to generate audio.

For picture loaders, the standard flow is: import image, compile BASIC, then export the tape file you want.

4. The Image Spectrumizer (SCR Converter) +

The Spectrumizer converts modern images into ZX-style screen graphics.

  • Standard ZX Spectrum: classic 256×192 screen with 8×8 attribute blocks.
  • Timex Hi-Color: 8×1 attribute detail for finer color control.
  • Mode 4 / 16-Color Bitmap: no attribute clash, more direct bitmap rendering.
  • Extended Hi-Res: larger 384×304 image mode for advanced conversions.

Use brightness, contrast, saturation, chunk size, and dithering to tune the result before export.

5. Simulating & Playing Tapes +

The Tape Play tab simulates a cassette loading process with border flashing and synthetic tape sounds.

  • Load a TAP file or let the app generate tape data from the current image.
  • Use Play, Pause, and Stop to control playback.
  • Change playback speed for standard, fast, or turbo loading.
  • The progress bar shows how far the tape block has loaded.

This is useful for previewing how your converted image or BASIC program behaves during loading.

6. Real WAV Audio Decoding +

The WAV Decode tab analyzes real tape audio and tries to rebuild tape blocks from the waveform.

  • Upload a .WAV file recorded from a Spectrum tape.
  • Tap Analyze & Draw to inspect and decode the audio.
  • The decoder follows pulse transitions and cassette-style signal timing.
  • Use Abort if the file is wrong or the analysis should stop.

For the best result, use clean audio with minimal noise and a stable tape capture.

7. Extended Graphic Modes +

The app supports more than the classic Spectrum screen layout.

  • Standard ZX Spectrum 256×192: 6,912-byte screen layout with 8×8 attribute clash.
  • Timex Hi-Color 256×192: 12,288-byte layout with 8×1 color detail.
  • Mode 4 / 16-Color Bitmap 256×192: 24,576-byte bitmap mode with no attribute clash.
  • Extended Hi-Res 384×304: 58,368-byte canvas for larger 16-color bitmap work.

The video mode choice also changes how the image buffer, dithering, tape export, and decoding logic behave.

8. Sinclair BASIC Compiler +

The BASIC Compiler turns numbered Sinclair BASIC lines into a tape-ready program block.

  • It tokenizes supported keywords like PRINT, LOAD, IF, GOTO, and POKE.
  • It writes a proper program header and data block for TAP output.
  • Optional image data can be appended as a screen block when enabled.
  • The program name and autostart line are taken from the BASIC Editor fields.

If you are building an image loader, a simple working pattern is: choose an image, write a BASIC loader line, then compile to .TAP.
