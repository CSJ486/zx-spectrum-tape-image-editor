User Manual Help & Guides
1. How to Code in Sinclair BASIC +

The BASIC Editor tab allows you to write real Sinclair BASIC programs. When compiled, the text is automatically tokenized into native ZX Spectrum bytecode.

  • Every line must begin with a Line Number (e.g., 10 PRINT "HELLO").
  • Use : to separate multiple commands on a single line (e.g., 10 BORDER 2: PAPER 0: INK 7: CLS).
  • The compiler auto-capitalizes keywords and handles tokenization (e.g., PRINT becomes token 0xF5).
  • Use the Load Example Program dropdown to see working templates.
2. Adding an Image to your BASIC Program +

You can bundle a picture with your game or software so it loads first like a classic title screen.

  1. Go to the BASIC Editor tab.
  2. Click Select PHOTO / IMAGE and choose a picture from your device.
  3. Ensure the Append Converted Image (Screen$) switch is ON.
  4. Write your code. The compiler will automatically insert a LOAD "" SCREEN$ command at the very start of your program if you haven't written one manually.
  5. Click Compile to .TAP. The resulting file will contain the Image Block first, followed by your BASIC Block.
3. Saving & Exporting to Tape (.TAP / .WAV) +

This tool supports generating both emulator-friendly data files and real audio cassettes.

  • .TAP Export: In the Spectrumizer or BASIC tabs, clicking Export/Compile will instantly download a clean digital .TAP file. These are perfect for emulators like Fuse or Spectaculator.
  • .WAV Audio Export: If you want to load your creation onto real hardware using a cassette deck or your phone's headphone jack, go to the Tape Play tab. Click Export Current tape to .WAV. This uses a synthesizer engine to write the distinct pilot tones and data pulses into a standard audio file.
4. The Image Spectrumizer (SCR Converter) +

The Spectrumizer tab converts modern photos into 256x192 retro graphics subject to "Attribute Clash" limits (only 2 colors allowed per 8x8 pixel block).

  • Fit Mode: Controls how the image scales. Smart Contain extends the edges to fill blank space smoothly.
  • Dithering: Mixes pixels to simulate more colors and shades. Turn this off for sharp cartoon-like art, or leave it on for photographs.
  • Monochrome: Forces a strict 1-bit black and white output which bypasses the attribute clash limit entirely, creating a crisp sketch look.
5. Simulating & Playing Tapes +

The Tape Play tab acts like a virtual cassette deck.

  • You can upload an existing .TAP file to watch it load, or play the image you just converted.
  • Clicking Play will simulate the tape loading process in real-time. You will hear the classic screeching sounds and see the border flashing accurately based on the T-State timings of a real ZX Spectrum CPU.
6. Real WAV Audio Decoding +

Have an audio recording of a real ZX Spectrum cassette? The WAV Decode tab can analyze the screeching noises and translate them back into an image.

  • It uses an offline Schmitt-trigger algorithm to detect the rising and falling edges of the audio wave to distinguish 0 bits from 1 bits.
  • Simply upload your tape's .WAV file, click Analyze, and watch it draw the screen data live.
